# Техническое задание: Telegram-бот для отслеживания подписок

## 1. Краткое описание

Создать личного Telegram-бота для учёта платных подписок на интернет-сервисы: ИИ-инструменты, видеосервисы, музыку, VPN, хостинги, игры и другие услуги.

Пользователь добавляет подписки, указывает стоимость, валюту, период оплаты, дату ближайшего списания, способ оплаты, ссылку на управление подпиской и заметки. Бот показывает ближайшие платежи, расходы за месяц и год, напоминает о скорых списаниях, позволяет отмечать оплату, приостанавливать, отменять и архивировать подписки.

Первая версия предназначена для одного владельца бота и не требует сложной панели управления или многопользовательского режима.

## 2. Цель проекта

**Цель:** создать Telegram-бота, который позволяет вести личный список подписок, контролировать даты будущих списаний и расходы, а также выдаёт результат в виде уведомлений, списков платежей и финансовых сводок.

## 3. Целевая аудитория

Основной пользователь — владелец Telegram-бота, который оплачивает несколько цифровых сервисов и хочет не забывать о регулярных списаниях.

Типичные категории:

- ИИ-сервисы;
- видеосервисы и музыка;
- VPN и прокси;
- облачные хранилища;
- хостинги и домены;
- программы и рабочие инструменты;
- игры;
- подписки на сайты и сообщества;
- другие регулярные платежи.

## 4. Принятые допущения

- Бот является личным: доступ разрешён только Telegram-пользователю с заранее указанным `ADMIN_TELEGRAM_ID`.
- База данных хранится локально на сервере или компьютере, где запущен бот.
- Основная валюта отчётов — рубли, но пользователь может добавлять подписки в любой валюте.
- В первой версии не требуется автоматическое получение курсов валют и интеграция с банками.
- Конвертация валют в общую сводку выполняется по вручную заданному курсу или не выполняется: суммы показываются отдельно по валютам.
- Напоминания приходят в Telegram.
- Бот работает постоянно через long polling или webhook. Для простой локальной установки рекомендуется long polling.

## 5. Основной сценарий работы

1. Пользователь открывает бота в Telegram.
2. Нажимает кнопку «Добавить подписку».
3. Вводит название сервиса.
4. Выбирает категорию.
5. Указывает стоимость, валюту и период оплаты.
6. Указывает дату следующего списания.
7. При необходимости добавляет способ оплаты, ссылку на управление подпиской и заметку.
8. Бот сохраняет подписку и показывает её карточку.
9. Перед ближайшим списанием бот отправляет напоминание.
10. После оплаты пользователь нажимает «Отметить оплаченной».
11. Бот записывает платёж в историю и автоматически рассчитывает следующую дату списания.
12. Пользователь в любой момент открывает сводку расходов, ближайшие платежи, категории или архив.

## 6. Архитектура проекта

### Тип приложения

Telegram-бот с локальной базой данных.

### Рекомендуемая архитектура

Модульный монолит.

Причина: проект личный, небольшой и не требует отдельного фронтенда, сложного сервера, микросервисов или очередей задач. Все функции можно поддерживать в одном Python-проекте.

### Основные модули

- `main` — запуск приложения, инициализация бота и планировщика.
- `config` — загрузка настроек из `.env`.
- `bot` — обработчики Telegram-команд, кнопок и диалогов.
- `services/subscription_service` — создание, изменение, удаление, архивирование и поиск подписок.
- `services/payment_service` — отметка оплаты, расчёт следующей даты, ведение истории.
- `services/report_service` — расчёт расходов, ближайших платежей и статистики.
- `services/reminder_service` — поиск подписок с приближающимся списанием и отправка уведомлений.
- `repositories` — работа с базой данных.
- `models` — модели базы данных.
- `keyboards` — кнопки Telegram.
- `utils` — форматирование валют, дат, текста, проверка ввода.
- `logger` — настройка безопасного логирования.
- `tests` — автоматические проверки ключевой логики.

### Взаимодействие модулей

```text
Telegram
   ↓
bot handlers
   ↓
services
   ↓
repositories
   ↓
SQLite database
```

Планировщик напоминаний работает параллельно с ботом:

```text
scheduler
   ↓
reminder_service
   ↓
subscription_service / repository
   ↓
Telegram notification
```

### Хранение настроек

Настройки хранятся в файле `.env`.

Пример параметров:

```env
BOT_TOKEN=...
ADMIN_TELEGRAM_ID=123456789
TIMEZONE=Europe/Istanbul
REMINDER_DAYS=1,3,7
DATABASE_URL=sqlite:///data/subscriptions.db
LOG_LEVEL=INFO
```

### Хранение данных

- база данных: `data/subscriptions.db`;
- резервные копии: `backups/`;
- логи: `logs/`;
- экспортируемые отчёты: `exports/`.

### База данных

Нужна.

Для первой версии использовать SQLite. Она проста, не требует отдельного сервера и подходит для одного пользователя.

### Фоновые задачи

Нужны.

Использовать встроенный планировщик, который раз в день проверяет:

- подписки с ближайшим списанием;
- подписки с просроченной датой;
- необходимость отправки повторного напоминания;
- актуальность даты следующего платежа.

### Логирование

Логи записываются в `logs/app.log`.

Записывать:

- запуск и остановку бота;
- ошибки обработки команд;
- ошибки базы данных;
- отправку напоминаний;
- создание и изменение подписок;
- отметку оплаты;
- ошибки при расчётах дат.

Не записывать в логи токен бота, персональные платёжные данные и полные ссылки, если они содержат приватные параметры.

## 7. Рекомендуемый стек

- язык: Python 3.12;
- Telegram-библиотека: `aiogram 3`;
- база данных: SQLite;
- ORM: SQLAlchemy;
- миграции базы: Alembic;
- планировщик задач: APScheduler;
- конфигурация: `.env` + `pydantic-settings`;
- валидация: Pydantic;
- тестирование: pytest;
- форматирование и проверка кода: Ruff;
- запуск: Python virtual environment;
- зависимости: `pyproject.toml` через `uv` или `pip`.

### Обоснование

Python и aiogram позволяют быстро создать удобного Telegram-бота с кнопками, диалогами и напоминаниями. SQLite не требует отдельного сервера базы данных. APScheduler достаточно для ежедневной проверки платежей. Такой стек удобно запускать на Windows 11, VPS или в WSL.

## 8. Функции первой версии

### 8.1. Обязательные функции

#### Функция: Добавление подписки

**Описание:**  
Пользователь создаёт новую подписку через последовательный диалог в Telegram.

**Вход:**

- название сервиса;
- категория;
- стоимость;
- валюта;
- период оплаты;
- дата следующего списания;
- способ оплаты;
- ссылка на управление подпиской;
- заметка.

**Выход:**  
Карточка созданной подписки.

**Критерий готовности:**  
Пользователь может добавить подписку, после чего она появляется в списке активных подписок и учитывается в отчётах.

#### Функция: Просмотр списка подписок

**Описание:**  
Бот показывает активные подписки в виде списка с краткой информацией.

**Вход:**  
Выбранный фильтр или категория.

**Выход:**  
Список подписок с названием, суммой, валютой, датой ближайшего списания и статусом.

**Критерий готовности:**  
Пользователь может посмотреть все активные подписки, только ближайшие платежи или подписки выбранной категории.

#### Функция: Карточка подписки

**Описание:**  
Пользователь открывает подробную информацию о конкретной подписке.

**Вход:**  
Выбор подписки из списка.

**Выход:**  
Карточка с полными данными:

- название;
- категория;
- сумма;
- валюта;
- период;
- дата следующего списания;
- способ оплаты;
- ссылка;
- заметка;
- статус;
- история последних платежей.

**Критерий готовности:**  
Пользователь может открыть карточку из списка и увидеть все сохранённые поля.

#### Функция: Редактирование подписки

**Описание:**  
Пользователь может изменить любое поле существующей подписки.

**Вход:**  
Новое значение выбранного поля.

**Выход:**  
Обновлённая карточка.

**Критерий готовности:**  
После изменения данных новая информация отображается в карточке, списках и отчётах.

#### Функция: Напоминания о списаниях

**Описание:**  
Бот отправляет сообщения о будущих платежах.

**Вход:**

- дата следующего списания;
- набор дней напоминаний, например 7, 3 и 1 день;
- статус подписки.

**Выход:**  
Telegram-уведомление.

Пример:

```text
Через 3 дня будет списание:

ChatGPT Plus
20 USD
Дата: 29 июня 2026
```

**Критерий готовности:**  
Бот отправляет только одно уведомление для каждой заданной точки напоминания и не напоминает об архивных или отменённых подписках.

#### Функция: Отметка оплаты

**Описание:**  
Пользователь нажимает кнопку «Оплачено».

**Вход:**

- подтверждение оплаты;
- при необходимости фактическая сумма;
- при необходимости комментарий.

**Выход:**

- запись в истории платежей;
- расчёт следующей даты списания;
- обновлённая карточка подписки.

**Критерий готовности:**  
После подтверждения платежа дата следующего списания автоматически переносится на следующий период.

#### Функция: История платежей

**Описание:**  
Для каждой подписки сохраняется история всех отмеченных оплат.

**Вход:**  
Выбор подписки.

**Выход:**  
Список платежей с датой, суммой, валютой и заметкой.

**Критерий готовности:**  
После нажатия «Оплачено» новая запись появляется в истории выбранной подписки.

#### Функция: Приостановка, отмена и архив

**Описание:**  
Пользователь может временно приостановить подписку, отменить её или убрать в архив.

**Вход:**  
Выбор действия.

**Выход:**  
Обновлённый статус подписки.

Статусы:

- `active` — активна;
- `paused` — приостановлена;
- `cancelled` — отменена;
- `archived` — в архиве.

**Критерий готовности:**  
Приостановленные, отменённые и архивные подписки не учитываются в ближайших платежах и не получают напоминания.

#### Функция: Сводка расходов

**Описание:**  
Бот показывает общую статистику по подпискам.

**Выход:**

- количество активных подписок;
- расходы за текущий месяц;
- прогноз расходов на следующий месяц;
- расходы за текущий год;
- разбивка по категориям;
- суммы отдельно по каждой валюте;
- ближайшие списания.

**Критерий готовности:**  
После добавления хотя бы одной подписки сводка корректно отображает её стоимость и дату.

#### Функция: Фильтрация и категории

**Описание:**  
Пользователь может группировать подписки по категориям и статусам.

**Базовые категории:**

- ИИ;
- работа;
- видео;
- музыка;
- VPN;
- хостинг;
- облако;
- игры;
- образование;
- другое.

**Критерий готовности:**  
Пользователь может выбрать категорию и увидеть только связанные подписки.

#### Функция: Экспорт данных

**Описание:**  
Пользователь может выгрузить данные по подпискам и платежам.

**Выход:**

- CSV-файл со списком подписок;
- CSV-файл с историей платежей.

**Критерий готовности:**  
По нажатию кнопки «Экспорт» бот отправляет пользователю файлы с актуальными данными.

#### Функция: Резервная копия базы

**Описание:**  
Бот создаёт копию базы данных по команде администратора.

**Выход:**  
Файл резервной копии SQLite.

**Критерий готовности:**  
Пользователь получает файл базы или копию, сохранённую в папке `backups/`.

## 12. Логика работы

1. Пользователь запускает бота командой `/start`.
2. Бот проверяет, совпадает ли Telegram ID пользователя с `ADMIN_TELEGRAM_ID`.
3. Если доступ разрешён, бот показывает главное меню.
4. При добавлении подписки бот последовательно собирает данные.
5. Каждый ввод валидируется до сохранения.
6. После успешной проверки данные сохраняются в SQLite.
7. Планировщик раз в сутки проверяет активные подписки.
8. Для каждой подписки рассчитывается количество дней до списания.
9. Если день совпадает с настройкой напоминания, бот отправляет уведомление.
10. При отметке оплаты создаётся запись платежа.
11. Для периодических подписок вычисляется новая дата списания.
12. Для подписок с периодом «вручную» бот просит пользователя указать следующую дату.
13. При построении отчётов бот учитывает только активные подписки.
14. При экспорте бот создаёт CSV-файлы и отправляет их в Telegram.
15. Все ошибки записываются в лог без чувствительных данных.

### Поведение при ошибках

**Пустой ввод:**  
Бот повторно просит заполнить поле или предлагает отменить действие.

**Неверный формат даты:**  
Бот объясняет формат и просит ввести дату повторно.

**Неверная сумма:**  
Бот принимает только положительные числа с точкой или запятой.

**Нет доступа к базе данных:**  
Бот сообщает о временной технической ошибке, а подробности пишет в лог.

**Подписка не найдена:**  
Бот возвращает пользователя к списку подписок.

**Файл экспорта уже существует:**  
Бот добавляет время к имени файла или перезаписывает только временный экспорт.

**Ошибка отправки сообщения:**  
Ошибка логируется, при следующей проверке напоминание не должно бесконечно дублироваться без контроля.

**Дата списания уже прошла:**  
Бот помечает подписку как просроченную и предлагает:

- отметить как оплаченную;
- изменить дату;
- отменить;
- отправить в архив.

## 15. Этапы реализации

### Этап 1. Основа проекта

- создать структуру папок;
- настроить виртуальное окружение;
- добавить зависимости;
- настроить `.env`;
- реализовать запуск бота;
- добавить ограничение доступа по Telegram ID;
- подключить SQLite.

**Результат:** бот запускается и отвечает на `/start`.

### Этап 2. Управление подписками

- создать модели базы;
- реализовать добавление подписки;
- реализовать просмотр списка;
- реализовать карточку подписки;
- добавить редактирование;
- добавить статусы подписок.

**Результат:** пользователь может полностью вести список подписок.

### Этап 3. Платежи и история

- реализовать отметку «Оплачено»;
- рассчитывать дату следующего списания;
- сохранять платежи в историю;
- показывать историю в карточке подписки.

**Результат:** платежи фиксируются, а даты обновляются автоматически.

### Этап 4. Напоминания

- подключить APScheduler;
- реализовать ежедневную проверку;
- реализовать защиту от повторных уведомлений;
- добавить настройку дней напоминаний.

**Результат:** бот своевременно сообщает о будущих списаниях.

### Этап 5. Отчёты и экспорт

- реализовать сводку расходов;
- добавить категории и фильтры;
- реализовать ближайшие списания;
- создать CSV-экспорт;
- добавить резервное копирование базы.

**Результат:** пользователь видит аналитику и может сохранить данные.

### Этап 6. Проверка качества

- добавить тесты расчёта дат;
- проверить обработку неверного ввода;
- проверить напоминания;
- проверить экспорт;
- проверить работу с архивными подписками;
- проверить запуск на Windows 11.

## 16. Критерии приёмки

Проект считается готовым, если:

1. Бот запускается на Windows 11 и получает токен из `.env`.
2. Посторонний Telegram-пользователь не получает доступ к данным.
3. Можно добавить подписку со всеми обязательными полями.
4. Можно изменить, удалить, приостановить, отменить и архивировать подписку.
5. Можно отметить оплату и увидеть её в истории.
6. Следующая дата списания рассчитывается правильно для ежемесячных, ежегодных, еженедельных и квартальных платежей.
7. Бот отправляет напоминания за заданное число дней.
8. Повторные напоминания не дублируются.
9. В сводке правильно считаются расходы по активным подпискам.
10. Можно отфильтровать подписки по категории и статусу.
11. Можно выгрузить список подписок и историю платежей в CSV.
12. Можно создать резервную копию базы.
13. Ошибочный ввод не ломает сценарий и сопровождается понятным сообщением.
14. В логах нет токена бота и приватных платёжных данных.
15. В `README.md` есть пошаговая инструкция по запуску на Windows 11.
